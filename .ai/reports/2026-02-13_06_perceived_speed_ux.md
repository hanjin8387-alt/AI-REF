# Perceived Speed / UX Report – 2026-02-13

## Baseline & Measurement
| 시나리오 | 현재 체감 | 목표 | 측정 방법 |
|---------|---------|------|----------|
| 앱 시작 → 첫 데이터 | ~2-3초 | ≤1.5초 | console.time + 캐시 히트 |
| 탭 전환 → 데이터 표시 | ~1-2초 (비캐시) | ≤0.5초 | 수동 체감 + 타이밍 |
| 레시피 추천 응답 | 3-10초 | 즉시 스켈레톤 + ≤5초 결과 | 타이밍 로그 |
| 재고 항목 삭제 반영 | ~0.5-1초 | 즉시 | 수동 체감 |
| 스캔 결과 대기 | 5-10초 | 즉시 상태표시 + ≤8초 | 진행 UI |

## Findings

### 🔴 Critical

#### PS-001: 스켈레톤/플레이스홀더 컴포넌트 전무
- **파일**: 탭 화면 전반
- **유형**: 체감속도
- **근거**: 데이터 로딩 시 빈 화면 또는 스피너만 표시. 레이아웃 시프트 발생.
- **영향**: 사용자가 "느리다"고 인식 (실제 로딩 시간 동일해도)
- **권장 조치**: `SkeletonCard` 컴포넌트 생성, 데이터 화면 3개 이상에 적용
- **예상 영향**: 체감 대기 -40%

### 🟡 Warning

#### PS-002: 낙관적 UI 미적용
- **파일**: 재고 삭제, 장보기 항목 추가/삭제
- **유형**: 체감속도
- **근거**: 서버 응답 대기 후에야 UI 갱신. 0.5-1초 지연.
- **권장 조치**: UI 즉시 반영 → 서버 확인 → 실패 시 롤백 (토스트 알림)
- **예상 영향**: 인터랙션 체감 즉각

#### PS-003: 레시피 추천 대기 시 진행 상태 없음
- **파일**: 레시피 탭 화면
- **유형**: 체감속도
- **근거**: Gemini 호출 3-10초 동안 단순 스피너. 진행 메시지/단계 없음.
- **권장 조치**: 단계별 메시지 ("재고 분석 중...", "레시피 생성 중...", "최적화 중...")
- **예상 영향**: 체감 대기 견딜 수 있음

#### PS-004: 스캔 업로드 진행률 미표시
- **파일**: 스캔 화면
- **유형**: 체감속도
- **근거**: 이미지 업로드 + Gemini 분석 동안 진행률 바 없음
- **권장 조치**: `onUploadProgress` 활용한 프로그레스 바 + 분석 중 애니메이션
- **예상 영향**: 불안감 감소

#### PS-005: 빈 상태에서 다음 액션 안내 부재
- **파일**: 탭 화면 (inventory, history)
- **유형**: 체감속도
- **근거**: 빈 상태 시 정적 텍스트만. 스캔/추가 버튼으로의 CTA 없음.
- **권장 조치**: 빈 상태에 컨텍스트 적합 CTA 버튼

### 🟢 Info

#### PS-006: 화면 전환 시 fade/slide 트랜지션 미설정
- 기본 네비게이션 트랜지션. 커스텀 트랜지션으로 프리미엄 느낌 강화 가능.

## Recommendations

### 체감속도 개선
1. `SkeletonCard` 공통 컴포넌트 생성 (GlassCard 기반 shimmer)
2. 재고 삭제·장보기 추가에 낙관적 UI 패턴 적용
3. 레시피 추천에 단계별 진행 메시지
4. 스캔 업로드에 진행률 바

### 실제 성능 연계
- 체감속도 개선은 실제 성능과 무관하게 적용 가능하나, 백엔드 캐시 개선과 조합 시 최대 효과
