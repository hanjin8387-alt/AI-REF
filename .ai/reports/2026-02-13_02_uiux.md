# UI/UX Report – 2026-02-13 (Feature Enhancement Cycle)

## Summary
- **검토 화면**: 5 탭 화면, 6 컴포넌트
- **발견 항목**: 🔴 Critical: 1 / 🟡 Warning: 5 / 🟢 Info: 2

## Findings

### 🔴 Critical

#### UX-001: 인터랙티브 요소에 `accessibilityLabel` 체계적 미적용
- **화면**: 전체 탭 화면 + `InventoryItemCard`, `RecipeCardStack`, `RoundButton`
- **카테고리**: 접근성
- **근거**: 모든 컴포넌트 파일 대상 `accessibilityLabel` grep 결과 일관성 없음
- **권장 조치**: Touchable/Pressable 요소에 컨텍스트 적합 label 추가. `accessibilityRole` 병기.
- **위험도**: 🔴

### 🟡 Warning

#### UX-002: 빈 상태(Empty State) 디자인 미통일
- **화면**: `inventory.tsx`, `history.tsx`, `alerts.tsx`
- **근거**: 빈 상태 렌더링 패턴이 화면마다 상이 (일부 텍스트만, 일부 CTA 포함)
- **권장 조치**: 공통 `EmptyState` 컴포넌트 추출, 아이콘+메시지+CTA 패턴 통일

#### UX-003: 에러 상태에서 재시도 UX 부재
- **화면**: API 호출 실패 시 전반
- **근거**: `http-client.ts`에서 에러 반환 후 화면에서 메시지만 표시
- **권장 조치**: 에러 상태에 "다시 시도" 버튼 + pull-to-refresh 안내

#### UX-004: 하드코딩된 색상값 다수
- **화면**: 탭 화면 및 컴포넌트 인라인 스타일
- **근거**: `'#1E293B'`, `'#00D084'`, `'#FF6B6B'` 등 Colors.ts 토큰 미사용
- **권장 조치**: Colors 토큰으로 교체

#### UX-005: 삭제/비가역 작업 확인 대화상자 불일치
- **화면**: 재고 삭제, 장보기 항목 삭제
- **근거**: 일부 삭제에만 `Alert.alert()` 확인 존재
- **권장 조치**: 모든 삭제에 통일된 확인 모달

#### UX-006: 로딩 상태 피드백 불일치
- **화면**: 스캔 업로드, 레시피 추천
- **근거**: 일부 화면은 ActivityIndicator, 일부는 커스텀 스피너, 일부는 없음
- **권장 조치**: 공통 `LoadingOverlay` 컴포넌트 도입

### 🟢 Info

#### UX-007: `GlassCard` 재사용 범위 확대 가능
- 일부 화면에서 직접 스타일로 유사 카드 UI 구현 중

#### UX-008: 라이트 모드 미지원 (백로그)
- `useColorScheme` 훅 존재하나 다크 전용

## Recommendations
1. `EmptyState`, `LoadingOverlay`, `ConfirmDeleteModal` 공통 컴포넌트 추출
2. Colors.ts에 semantic 토큰 확장 (예: `colors.error`, `colors.success`, `colors.textSecondary`)
3. 접근성 린트 규칙 도입 (eslint-plugin-react-native-a11y)
